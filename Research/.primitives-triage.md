# Primitives Package Triage for swift-sample-primitives

## Triage Summary

| Package | Verdict | Rationale |
|---------|---------|-----------|
| **swift-comparison-primitives** | SKIP | `Comparison` enum (less/equal/greater) is not the same concept as our `Comparison<T>` (statistical). No naming conflict (enum vs generic struct). `Comparison.Protocol` (~Copyable Comparable) not needed — Sample uses stdlib `Comparable`. |
| **swift-algebra-monoid-primitives** | SKIP | Accumulator IS a commutative monoid, but the package provides value witnesses, not protocol conformances. Can't use value witnesses for conditional extensions. Document monoidal structure in comments per first-principles recommendation. |
| **swift-algebra-module-primitives** | SKIP | `Algebra.Module<Int, T>` is mathematically the right structure for "T / Int". But it's a value witness — can't power `extension Sample where Element: ...`. We need a protocol (`Averageable`), not a witness value. |
| **swift-algebra-field-primitives** | SKIP | Provides field division but for field elements dividing by other field elements, not "divide T by Int". Wrong abstraction level. |
| **swift-numeric-primitives** | SKIP | `Numeric.Transcendental` provides sqrt but `Double.squareRoot()` is stdlib. No protocol for "divisible by Int" exists here. |
| **swift-dimension-primitives** | SKIP | Geometric coordinate dimensions (X/Y/Z), not physical units. `Measure<2>` = area, not seconds². Not relevant to variance/seconds² problem. |
| **swift-scalar-primitives** | SKIP | Package exists but is empty (no source files). |
| **swift-index-primitives** | SKIP | Phantom-typed indices. Percentile indexing uses plain Int internally. |
| **swift-cardinal-primitives** | SKIP | Phantom-typed counts. Sample.count is plain Int. |
| **swift-ordinal-primitives** | SKIP | Ordinal positioning. Percentile rank is computed from Double, not Ordinal. |
| **swift-sequence-primitives** | SKIP (v1) | Sample exposes `.values: [T]` which is already a Sequence. No need to conform Sample itself to Sequence.Protocol — over-engineering for a statistics type. |
| **swift-collection-primitives** | SKIP (v1) | Same reasoning. Users access statistics, not individual elements. `.values` gives Collection access when needed. |
| **swift-array-primitives** | SKIP | Sample uses stdlib `[T]`. No need for custom array types at these sample sizes (10-10,000 elements). |
| **swift-buffer-primitives** | SKIP | Ring buffer for sliding-window statistics would be a future addition, not v1. |
| **swift-time-primitives** | SKIP | Provides calendar types (Year, Month, Hour), not `Duration`. Swift stdlib provides `Duration` since 5.9. |
| **swift-clock-primitives** | SKIP | ContinuousClock is stdlib. `measure()` lives at L3 (swift-tests), not L1 (sample-primitives). |
| **swift-test-primitives** | N/A | Already used by swift-tests. Not a dependency of sample-primitives (different layer). |
| **swift-source-primitives** | N/A | Already used by swift-tests. Not relevant to sample-primitives. |
| **swift-formatting-primitives** | SKIP | Presentation lives at L3, not L1. |
| **swift-property-primitives** | SKIP (v1) | Property.View accessor pattern could be used for `.statistics.mean` etc., but adds dependency weight for no clear benefit at this stage. |
| **swift-witness-primitives** | SKIP | Witness infrastructure for value witnesses. Not needed if we're using protocols + conditional extensions. |
| **swift-tagged-primitives** | SKIP | Namespace/identity infrastructure. sample-primitives defines its own namespace. |
| **swift-equation-primitives** | SKIP | ~Copyable equality. stdlib Equatable suffices. |
| **swift-hash-primitives** | SKIP | Hashable infrastructure. Not needed for statistics. |
| **swift-finite-primitives** | SKIP | Finite sets. Not relevant. |

## Conclusion

**swift-sample-primitives should have ZERO dependencies on existing primitives packages.**

All required infrastructure comes from the Swift standard library:

| Need | Source |
|------|--------|
| `Comparable` | stdlib |
| `AdditiveArithmetic` | stdlib |
| `Array<T>` | stdlib |
| `Double`, `UInt64`, `Int` | stdlib |
| `Duration` (for typed extensions) | stdlib (Swift 5.9+) |
| `squareRoot()` | stdlib `FloatingPoint` |
| `Sendable` | stdlib |
| `/ Int` on Duration | stdlib |

The only new protocol is `Averageable` (AdditiveArithmetic + `/ Int`), which is specific to the statistics domain and belongs in swift-sample-primitives itself.

This is the cleanest possible result: a zero-dependency primitives package providing pure mathematical types. Analogous to Haskell's `statistics` depending only on `base` + `vector`.

### Future Integration (Trait-Gated, Per [MOD-014])

If ecosystem integration is desired later:
- `Sample` could conform to `Sequence.Protocol` via a trait-gated integration target
- `Accumulator` could provide `Algebra.Monoid.Commutative` witness values via integration target
- `Metric` could use `Property.View` accessor pattern via integration target

These are all additive, non-breaking, and can be done after v1 ships.
